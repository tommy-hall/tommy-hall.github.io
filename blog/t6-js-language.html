Javascript Language

( ) - Parentheses:

  • Hold paramater names when defining a function and hold argument values when calling a function. E.g. Defining the function: golf(par, score) and calling the function: golf(4, 3). Note: If parameters are not required, empty parenthesis must still be used when defining and calling a function.
  • Surround conditional statements in some functions. E.g.if(it snows tomorrow)wear boots.
  • Determine the order of operations in mathematical equations. E.g.X = 4 + (3 x 2)

[ ] - Brackets:

  • Hold values in an array (each one separated by a comma) and hold the index number when accessing an array. E.g.basket [apple, banana, lemon] and basket[2] = lemon

{ } - Braces:

  • Holds KEY:VALUE pairs when defining an object (each pair separated by a comma). E.g.Basket {apple: large, banana: small}
  • Hold blocks of code within functions (each statement separated by a semicolon). E.g.make drink {pour water; get cup}

'Single quotes'

  • Used to surround strings of English language text to let the browser know it is not code.
  • The only rule is that the opening and closing quotations MUST be the same. This means you can use the opposite pair within your string and they will appear in the actual text on the page. E.g. "Press the 'contact' button!" = Press the 'contact' button!

"Double quotes":

  • Double quotes do exactly the same job as single quotes, so it is down to personal preference which you choose to use as a developer.
  • As shown above, you would also use double quotes in conjunction with single quotes where you have a need to display a string within the string you have already declared, or where you need to exit the string in order to display some other code such as in some instances when you are using JavaScript to alter HTML elements of a website.
